# AI-Privacy Sentinel

`AI-Privacy Sentinel` — это браузерное расширение на `Manifest V3`, которое снижает риск случайной отправки конфиденциальных данных в браузере: в обычных полях ввода и в LLM-чаты, такие как ChatGPT, Claude и Gemini.

Расширение локально анализирует текст прямо на странице, ищет возможные секреты и PII, подсвечивает рискованные поля ввода и при необходимости блокирует отправку сообщения, пока пользователь не замаскирует данные или не отменит действие.

## Возможности

- Полностью локальный анализ без внешних сетевых запросов.
- Поиск email-адресов.
- Поиск распространённых форматов API-ключей:
  - `sk-`
  - `ghp_`
  - `xoxb-`
  - `AIza`
- Поиск JWT-токенов.
- Поиск номеров банковских карт с проверкой по алгоритму Луна.
- Автоматическая маскировка через заглушки вроде `[HIDDEN_CARD]`.
- Поиск динамически появляющихся полей ввода через `MutationObserver`.
- Изолированный UI через `Shadow DOM`, чтобы стили сайта не ломали интерфейс расширения.
- Перехват отправки сообщения для типовых кнопок в формах и LLM-интерфейсах.

## Логика детекции

Детектор анализирует текст и ищет:

- email-адреса
- API-ключи с известными префиксами
- JWT-токены в формате `header.payload.signature`
- кандидаты на номера банковских карт длиной `13-19` цифр, в том числе с пробелами и дефисами

Номер карты считается валидной находкой только в том случае, если проходит проверку по алгоритму Луна.

Маскировка выполняется через `maskText(text, findings)`, которая заменяет найденные значения на заглушки:

- `[HIDDEN_EMAIL]`
- `[HIDDEN_API_KEY]`
- `[HIDDEN_JWT]`
- `[HIDDEN_CARD]`

## Установка

1. Открой Chrome или любой Chromium-браузер.
2. Перейди на страницу `chrome://extensions`.
3. Включи `Developer mode`.
4. Нажми `Load unpacked`.
5. Выбери папку проекта:

```text
/Users/dk/Documents/New project
```

6. Открой один из поддерживаемых LLM-сайтов.
7. Вставь или введи тестовый секрет и проверь, что:
   - поле получает красную предупреждающую рамку
   - появляется защитный UI
   - отправка блокируется, пока текст не будет замаскирован или действие не будет отменено

## Как это работает

1. `content.js` отслеживает динамически добавляемые поля ввода на странице.
2. При событиях `input` и `paste` текст текущего поля проверяется через `detector.js`.
3. Если найдены чувствительные данные, поле визуально помечается как рискованное.
4. При нажатии на кнопку отправки расширение повторно анализирует активное поле.
5. Если секреты всё ещё присутствуют, отправка блокируется и показывается модальное окно внутри `Shadow DOM`.
6. Пользователь может выбрать `Замаскировать и отправить` или `Отмена`.

## Приватность и безопасность

- Весь анализ выполняется локально.
- Расширение не использует внешний backend и сторонние API.
- В `manifest.json` указаны только минимально необходимые разрешения:
  - `storage`
  - `activeTab`

## Текущие ограничения

- Детекция эвристическая, поэтому возможны ложные срабатывания и пропуски.
- LLM-сайты регулярно меняют DOM-структуру, поэтому селекторы со временем могут потребовать обновления.
- Текущая версия покрывает базовый набор паттернов, но не все существующие типы токенов и секретов.
- Полноценную рантайм-проверку в этом окружении выполнить не удалось, потому что `node` недоступен в текущем shell.
