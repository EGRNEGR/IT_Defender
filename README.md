Что делает эта функция
Защищает Ваши ПД от распространения в сети. Может автоматически заменять чуствительную информацию или предупреждать пользователя / запрещать отправку. Работает с Почтами, номерами телефонов, ключами API, ФИО с датой рождения и т.д.
Расширение проверяет ссылки до перехода по ним и пытается остановить переход на известные вредоносные адреса.

Проверка срабатывает, когда вы:

нажимаете на ссылку
вставляете URL в поле
переходите на сайт напрямую
вручную запускаете проверку из popup
Если хотя бы один источник угроз помечает адрес как вредоносный, расширение показывает предупреждение и может заблокировать переход.

Как это работает
Расширение проверяет адрес по нескольким уровням:

локальная база фишинговых URL, доменов и IP
VirusTotal
PhishTank
OpenPhish
AbuseIPDB для IP-адресов
Логика простая:

если любой источник подтвердил угрозу, адрес считается опасным
если никто не подтвердил угрозу, переход разрешается
Что увидит пользователь
Есть два основных сценария.

Адрес безопасный
Ничего не происходит, либо в popup видно, что угроз не найдено.

Адрес подозрительный или вредоносный
Открывается страница предупреждения с:

причиной блокировки
списком источников, которые сработали
безопасными действиями
Что делать на странице предупреждения
На странице предупреждения доступны действия:

Назад
Рекомендуемое действие. Возвращает на предыдущую страницу.

Скопировать URL
Копирует адрес без перехода на него.

Сообщить о ложном срабатывании
Используйте, если уверены, что сайт безопасен, а блокировка ошибочная.

Всё равно продолжить
Доступно только если администратор разрешил обход.
Может потребоваться указать причину. Это действие записывается в журнал.

Как пользоваться popup
Во всплывающем окне расширения можно:

увидеть текущий домен
посмотреть итоговый вердикт: разрешить или блокировать
посмотреть, какие источники сработали
вручную нажать Проверить сейчас
Что означают настройки
Если вы обычный пользователь, обычно их менять не нужно. Но смысл такой:

Включить L1 gate
Включает проверку ссылок вообще.

Блокировать при вредоносном вердикте
Если включено, опасные ссылки реально блокируются.

Разрешить пользовательский обход
Позволяет вручную продолжить переход несмотря на предупреждение.

Требовать причину обхода
Просит указать, зачем вы всё равно переходите.

Подозрительные TLD
Список доменных зон повышенного риска.

Блокировать по сигналу подозрительного TLD
Делает такие доменные зоны поводом для блокировки.

Что важно понимать

Расширение не гарантирует 100% защиту.
Возможны ложные срабатывания.
Если страница предупреждения открылась, безопаснее нажать Назад.
Обход стоит использовать только если вы точно понимаете риск.
Коротко

Нажали ссылку -> расширение проверило адрес.
Нашло угрозу -> показало предупреждение.
Не нашло угрозу -> переход разрешён.
Если разрешён обход -> можно продолжить, но это фиксируется.
